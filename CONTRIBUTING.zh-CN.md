# 贡献指南

[English](CONTRIBUTING.md)

感谢你帮助改进 Family Doctor Skills。因为这个项目涉及健康和急救分流，贡献时应优先考虑安全、清晰和可追溯，而不是追求覆盖所有医学知识。

## 核心原则

- 分流要保守。
- 出现红旗信号时，应尽快引导用户进入现实医疗系统。
- 不添加最终诊断或处方决策。
- 使用普通用户在紧张状态下也能看懂的语言。
- 保留或增强对高风险人群的警示。
- 医学说法尽量使用权威来源支持。

## 欢迎贡献的内容

- 更清晰的分流表达
- 更好的红旗信号列表
- 儿童、老人、孕妇、过敏、慢病、用药安全等内容优化
- 不同国家和地区的急救电话、就医路径说明
- 更安全的输出模板
- 来自公共卫生、急救、毒物控制或专业医学组织的来源更新

## 医疗内容规则

不要添加下面这类建议：

- 鼓励用户延误急诊或急救
- 提供处方药剂量
- 建议普通病毒症状自行使用抗生素
- 建议用户自行停用或更改处方药
- 把孕妇、婴幼儿、老人、免疫低下、抗凝药使用者、严重过敏人群当作普通低风险情况处理
- 推荐不安全的烫伤、伤口、中毒或退烧偏方

如果不确定，选择更安全的升级处理路径。

## 来源要求

涉及医学或安全内容的修改，请尽量附带来源。推荐来源包括：

- CDC、中国疾控、卫健委或其他公共卫生机构
- American Red Cross、St John Ambulance 等急救组织
- Poison Control 或本地毒物控制机构
- ACOG、AAP、AHA、NICE 等专业组织
- 政府急救或公共卫生资源

不要把无来源博客、社交媒体、营销页面或个人经验作为医学建议基础。

## Pull Request 检查清单

提交 PR 前请检查：

- 修改仍然保持保守分流。
- 红旗信号仍然出现在居家建议之前。
- 高风险人群被更谨慎地处理。
- 用药建议没有越过安全边界。
- 文字在紧张状态下也清楚易懂。
- 重要医学修改附有来源。
- 需要时同步更新中英两个版本。

## Issue 反馈

报告安全问题时，建议包含：

- 用户提问或场景
- skill 的回答，如果有
- 为什么这条建议可能不安全或不完整
- 你建议的更安全表达
- 可用的权威来源链接

安全相关 issue 优先级高于普通功能请求。
