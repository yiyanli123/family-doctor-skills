# 信任与安全

[English](TRUST.md)

家庭医生 Skills 的定位是：**以安全为核心、参考权威来源设计的居家分流工具**。它们帮助用户判断下一步该做什么，但不替代医生、急救服务、诊断、处方或本地医疗判断。

## 为什么用户可以建立信任

### 1. 先做保守分流

这些 skills 会先检查急症红旗信号，再给居家处理建议。如果症状提示危险，回答应优先引导用户拨打急救电话、去急诊、去 urgent care，或当天联系医生。

### 2. 医疗边界清楚

这些 skills 不会：

- 做最终诊断
- 开处方
- 建议自行使用抗生素或处方药
- 建议用户自行停用或更改处方药
- 鼓励用户延误急诊或急救
- 把高风险人群当作普通低风险情况处理

### 3. 对高风险人群更谨慎

遇到下面人群时，skills 会降低就医阈值：

- 婴幼儿和儿童
- 老人
- 孕妇或产后人群
- 有严重过敏或过敏性休克史的人
- 哮喘、慢阻肺、心脏病、糖尿病、肾病、肝病患者
- 癌症治疗、移植用药、大剂量激素或其他免疫低下人群
- 使用抗凝药或多药联用的人

### 4. 内容可审查

项目把指导内容拆分成可阅读的 reference 文件，例如：

- `red-flags.md`
- `children.md`
- `older-adults.md`
- `pregnancy-allergy-high-risk.md`
- `medication-safety.md`
- `first-aid-injuries.md`

这让项目不是黑箱。用户和贡献者可以逐条 review、修正和改进。

### 5. 参考权威来源

这些 skills 的安全分流模式参考了公开的急救、公共卫生和医学安全资料，例如：

- CDC
- American Red Cross
- Poison Control
- ACOG
- 中国疾控、卫健委、急救中心等本地公共卫生/急救资源

对于可能随时间变化的信息，例如药物剂量、疫苗、传染病政策、召回、本地急救路径等，skills 会要求 Codex 查阅最新权威来源。

## “可靠”不代表什么

可靠不代表 skill 永远正确。这里的可靠指的是项目设计上尽量做到：

- 避免过度自信的医学判断
- 不确定时倾向更安全的升级处理
- 让内容可检查、可改进
- 遇到危险情况时，让用户尽快进入现实医疗系统，而不是继续聊天

## 如何判断一次回答是否可信

可信的回答通常应该：

1. 先识别红旗信号。
2. 只追问必要信息。
3. 给出明确下一步。
4. 说明不要做什么。
5. 说明什么时候就医。
6. 对高风险人群更保守。
7. 不假装能隔空确定诊断。

如果回答建议急诊、急救或当天就医，用户应优先执行现实世界的医疗行动，而不是继续聊天。

## 如何报告问题

如果你发现下面情况，请提交 issue：

- 可能延误急救的建议
- 不安全的用药建议
- 对儿童、老人、孕妇、过敏、慢病或免疫低下人群缺少警示
- 过期或缺少来源支持的医学说法
- 在高风险场景中可能被误解的模糊表达

报告时建议附上：用户提问、skill 回答、你看到的风险，以及可用的权威来源。
